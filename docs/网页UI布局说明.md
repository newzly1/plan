# 巴厘岛及周边 · 选点投票网页 — UI 布局说明

> 全部样式在 [style.css](../webpage/style.css)，由 [build.py](../webpage/build.py) 内联进 `index.html`。本文只讲**视觉与版式**；数据/交互/部署见 [网页整体结构说明.md](网页整体结构说明.md)。

---

## 一、设计基调

**「热带海洋 · 田野图鉴」杂志风**——像一本旅行手册内页，而非常规 App。贯穿全局的取向：

- **去卡片化**：景点、组合、章节都不用圆角卡片和阴影，而用**细墨线分隔**（`--rad:0px` 全站直角），像杂志栏目线。
- **单栏窄版心**：全站 `max-width:760px` 居中，移动优先，手机上就是一条竖屏长卷。
- **三种字体分工**：
  - 衬线宋体（`Trip Serif`）——只用于大标题（Hero、章名、景点名、组合名）；
  - 等宽字体（mono）——所有小标签/编号/数据（如 `A1`、`FIELD NOTES`、价格数字）；
  - 黑体（PingFang 等）——正文。
- **双主题**：`石灰岩浅色`（骨白青底 `#F0F5F3` + 深墨绿字）与 `火山深色`（深底 `#0D1A17`），跟随系统 `prefers-color-scheme`，也支持 `data-theme` 手动覆盖。
- **强调色**：深海绿 `--pine`（必去/主色）+ 珊瑚橙 `--brass`（点缀/编号/警示）。

## 二、从上到下的布局

### ① Hero 封面区（杂志式）
- 顶部 `masthead`：左 `INDONESIA`、右 `FIELD NOTES · 2026`，等宽大字距。
- 居中标题组：36px 短横线 + 宋体大标题（`clamp(30–44px)`）+ 拉开字距的副题 + 一行 `facts`（日期/天数/人数/DPS）。
- `hero-media` 大图：竖构图 `4/4.4`，宽屏变 `16/10`；下方 `cap-line` 图注（左景点名、右橙色 `Fig. 01`）。
- 底部 `howto` 使用说明 + `index` 章节导航（第一…第五章，`flex:1 0 30%` 三列细线网格）。

### ② 章节 chapter（第一章 最热门 + 各地区章 + 第六章 组合建议）
- 顺序：**第一章＝最热门景点**（4 个招牌），其后按地区依次（巴厘 / 佩尼达 / 宜珍 / 龙目）；整区只有热门景点的科莫多、布罗莫不单独成章；**第六章＝组合建议**（正文最后，也进 Hero 导航）。
- `chap-head` 用 **1.5px 墨线顶边**开章；`chap-meta` 左宋体章名、右橙色 **`第 N 章`** eyebrow。
- `chap-data` 一行元数据（第一章为「多数人必去」；地区章为 tag / 天数 / 预算，小圆点分隔）+ 灰色 `chap-desc` 描述段（第一章无描述段）。

### ③ 景点 spot（文章式，全站主体）
每个景点是一篇「短文」，细线顶边分隔，含：
- **spot-head**：左等宽 `A1` 编号 + 右宋体中文名（带 ★）+ 英文小注。
- **tags**：`· 文化 · 梯田` 橙点前缀标签。
- **gallery**：横向滑动图组，图 `4/3`，`cursor:zoom-in` 可点开灯箱。
- **video plate**：`16/9` 视频板，中心圆形播放钮 + 左上「▶ 点击播放」毛玻璃徽标 + 底部渐变字幕。
- **spot-meta**：细线顶边的「时间 / 人均」两项。
- **notes**：可折叠的「亮点·注意·串联」，带旋转箭头 `chev`。
- **投票区 vote**：`repeat(3,1fr)` 三个方角文字钮「必去/可去/略过」，选中态填色（必去=深绿、可去=墨黑、略过=浅灰）。
- **钢印 stamp**：投票后右上角出现一枚旋转 -13° 的圆形单色印章（如「必去」），带弹入动画。

### ④ 组合线路 combos
- 细线列表，左侧**宋体大编号 ①**（橙色 30px），右侧线路名 + 区域/跨岛/天数（含总天数与分区明细）/人均。宽屏变两列网格。
- ①–④ 为 9–10 天全程主线，⑤–⑦ 为 6 天精简版（仅能走 6 天的队员）。所有线路均含佩尼达一日游。
- **可点击选择**：点击任一线线卡片即选为你的主线偏好，再点一次取消；每张卡片右上角有 `选为我的主线` 按钮式标签（hover 变绿、选中绿底白字变 `✓ 已选为主线`），整卡 hover 有浅色背景反馈。

### ⑤ 折叠区 folds（价格 / 注意事项）
- 用细墨线**方框包围**的 `<details>`；价格表用等宽右对齐数字（`tabular-nums`）。

### ⑥ 页脚 footer
- 墨线顶边，小字灰色的图片署名 + 免责 + 等宽大字距的「为你此行制作」。

---

## 三、固定层 / 浮层（叠在内容之上）

- **底部状态栏 bar**（`position:fixed` 全宽墨线底条）：两行——上行「我的清单 ›」+ 橙色 `你选 N/14 · 群 N/6 已投`（你已选景点数 / 群里已投票人数）；下行 `bar-sub` 显示群体最爱/填名提醒（`warn` 态变橙）。
- **我的清单 sheet**：底部弹起的半屏浮层（`align-items:flex-end` + 上滑动画 + 背景毛玻璃遮罩），内含名字输入、我的必去/可去清单、**实时汇总面板 tally**（排行榜 + 组合票数），底部 sticky 的「提交我的选择（绿）/ 清空所有标记」按钮。
- **灯箱 lb**：近全黑遮罩居中大图，右上关闭钮。
- **toast**：底部居中的墨色小提示条，`nudge` 态变绿且可点。

---

## 四、响应式与动效

- **断点 720px**：Hero 图变宽幅 `16/10`、组合线路变两列。
- **入场动画**：`.spot / .combo` 滚动进入视口时上移淡入（`IntersectionObserver`）；`prefers-reduced-motion` 时全部关闭。
- **滚动条**：横向图廊/速览一律隐藏滚动条 + `scroll-snap` 吸附。

**一句话概括**：一条 760px 窄版心的竖向长卷，用**细墨线分栏 + 宋体标题 + 等宽标签 + 深绿/珊瑚橙点缀**营造「旅行图鉴」质感；主体是一篇篇文章式景点（图廊 + 视频 + 三键投票 + 钢印），底部常驻状态栏唤起半屏「我的清单」浮层完成投票与实时汇总。
