# 巴厘岛及周边 · 6 人选点投票网页 — 整体结构说明

> 一句话：**无后端的单文件自包含页面**。6 人看图/看视频，为每个候选景点投「必去 / 可去 / 略过」，供出行参考；仅投票的实时汇总用到 CloudBase 云数据库。核心产物是 `webpage/index.html`（外壳）+ `webpage/images/*.webp`（约 60 张图），两者放同一目录即可完全离线打开。

---

## 1. 架构 —— 构建生成式

手写的源码由 [build.py](../webpage/build.py) 组装成一个 `index.html`。这是一套**静态站点生成器**结构，内容 / 样式 / 交互彻底分离到不同文件：

```
content.py（内容） ─┐
style.css（样式）    ├─→ build.py ─→ index.html（自包含外壳）
app.js（交互）       │              + images/*.webp（从 media.json 解码）
tally.js/identity.js ┘
media.json / videos.json / credits.json（媒体与素材数据）
```

- [build.py](../webpage/build.py) **只用 Python 标准库**，`python3 build.py` 一条命令重新生成 `index.html` 和 `images/`。
- CSS 与 JS（`tally.js`、`identity.js`、`app.js`）全部内联进 `<style>` / `<script>`；只有 `cloudbase.js`（vendored SDK）作为独立文件引用。
- 图片来自 `media.json`（base64，约 8.85 MB），构建时解码写成 `images/<key>.webp`，页面里用相对路径 `<img src="images/A1a.webp">` 引用。图片路径只取决于景点 key，不受文字/样式改动影响。

## 2. 内容数据模型 —— [content.py](../webpage/content.py)

全部文字的**唯一来源**（从 `印尼旅行项目投票清单.md` 转写而来），结构化为 Python dict/list：

| 变量 | 内容 |
|---|---|
| `META` | 标题、副题、使用说明、汇率注记 |
| `REGIONS` | 6 个地区 A–F（巴厘本岛 / 佩尼达三岛 / 科莫多 / 布罗莫 / 宜珍 / 龙目+吉利） |
| `ITEMS` | **14 个景点**。每个含 `id, region, zh/en, tags, time, price, feature/caution/coupling（亮点/注意/串联）, video_q, subspots[]` |
| `HIGHLIGHTS` | 第一章「最热门景点」成员（4 个 item id，有序） |
| `COMBOS` | 组合线路 ①–④（9–10 天的主线方案） |
| `PRICES` | 参考价格表 4 组 |
| `NOTES` | 全程通用注意事项 8 条 |

每个 subspot 带英文检索词 `q`，作为图片抓取管线的输入。

## 3. 页面 DOM 结构（对应 [build.py](../webpage/build.py) 各函数）

自上而下：

1. **`hero()`** — 封面（大标题 + Kelingking 主图 + 章节导航「第一…第五章」，由 `spot_chapters()` 生成，与正文一一对应）
2. `<main>` 内：
   - **`chapters_html()`** — 按 `spot_chapters()` 顺序渲染景点章：**第一章＝最热门景点**（`HIGHLIGHTS` 成员），其后为**各非空地区章**（`REGIONS` 中仍有非热门景点的地区）。整区景点全为热门的地区（科莫多/布罗莫）不成章；章名 eyebrow 用「第 N 章」；每章内是景点卡 `spot()`。
   - **`combos()`** — 组合线路 ①–④（不编号）
   - **`prices()` / `notes_sec()`** — 可折叠的价格表 / 注意事项
   - **`footer()`** — 图片署名、免责声明
3. **`mylist()`** — 底部固定状态栏 + 「我的清单」浮层 + 灯箱 + toast

每个景点卡 `spot()` 含：图片画廊、哔哩哔哩视频板、亮点/注意/串联详情，以及**「必去 / 可去 / 略过」投票按钮**。

## 4. 交互层 —— [app.js](../webpage/app.js)

纯 `localStorage` 的 IIFE。主要功能：

- **投票** — 事件委托处理 `.v` 按钮；`votes` 存进 localStorage（键 `bali_votes`），并触发 `cloud.syncMine()` 云同步。
- **「我的清单」浮层** — 填名字、选主线偏好（combo）、看自己的必去/可去清单、**实时汇总面板**、「一键复制汇总」（生成可粘贴到群里的文本）、清空。
- **底部状态栏** — 显示 `你 N/14 · 群 N/6`，以及群体最爱 Top3 / 未填名警告等四种状态。
- **视频内联播放** — 哔哩哔哩 iframe，一次只播一个。
- **灯箱、toast、首次投票的填名轻提示、滚动出现动画**。

## 5. 云同步与汇总层

localStorage 只是各自的本地留存；6 人的合并汇总走 CloudBase 云数据库：

- **[cloudbase.js](../webpage/cloudbase.js)** — vendored 的 CloudBase Web SDK **v2.31.0**（同源部署，不依赖 CDN）。
- **[app.js](../webpage/app.js) 的 `cloud` 模块** — 匿名登录 → 把自己那一条 upsert 进集合 `votes`（800ms 防抖）→ 拉取全量。
- **[identity.js](../webpage/identity.js)** — 把名字归一化后派生 docId（`u_<base64url>`）；换设备/撞名时把本地与云端 **merge**（按 `must>maybe>skip` 取更强的一方，防丢票）。
- **[tally.js](../webpage/tally.js)** — 纯函数汇总算法。**加权分 = 必去×2 + 可去×1** 给景点排行，组合线路单独统计；浏览器与 `node --test` 共用。
- **兜底** — SDK 未加载/断网时，汇总面板降级为「云同步暂不可用」，本地投票与「一键复制」照常可用。

汇总刷新时机：打开浮层 / 投完票 / 点「刷新」/ 标签页重新可见 / 每 45 秒。

## 7. 辅助脚本（一般用不到）

大量 Python 脚本（`fetch_*.py`、`apply_review.py` 等）属于**图片抓取管线**，仅在换图/重抓时使用。日常改内容/样式/交互，只需跑 `build.py`。

**单测**：`node --test webpage/test/tally.test.js`（零依赖，Node 内置）。

---

**一句话概括**：Python 构建的单文件静态页面 + CloudBase 云数据库做轻量实时汇总。改文字改 `content.py`，改样式改 `style.css`，改交互改 `app.js`，改算分改 `tally.js`，然后 `python3 build.py` 重新生成即可。
