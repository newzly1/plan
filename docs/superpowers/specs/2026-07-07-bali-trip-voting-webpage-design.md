# 巴厘岛 6 人投票参考网页 · 设计文档（Spec）

- **日期**：2026-07-07
- **来源文档**：`印尼旅行项目投票清单.md`
- **交付形态**：claude.ai Artifact（单页、移动优先、中文）

---

## 1. 背景与目标

6 人 10/2–10/11（约 9–10 天）赴印尼、围绕巴厘岛出行，路线未定。现有 `印尼旅行项目投票清单.md`
按大区 A–E 列出可选项目，但**只有文字、无图无视频**，成员难以判断每个景点"到底长什么样、是干嘛的"。

**目标**：把该清单做成一个网页，每个景点配**真实照片 + 代表视频**，并支持成员在页面上做本地投票、
一键复制汇总粘到微信群，从而帮助 6 人高效选出想去的地方与主线。

**成功标准**：成员在手机上打开一个链接，能看图/看视频看懂每个点，标记"必去/可去/无所谓"，
复制出一段带名字与主线偏好的汇总文字。

---

## 2. 交付形态与技术约束

- **形态**：作为 **claude.ai Artifact 发布**，得到一个 claude.ai 网址，微信直接分享、手机打开。
- **网络前提**：6 人**均可访问 YouTube / Google**（已确认）。
- **Artifact CSP 约束（决定实现方式）**：
  - 禁止任何外部主机请求（外链图片 / CDN / 外部字体 / iframe / fetch 全部被拦）。
  - ⇒ **图片必须内嵌为 data URI**（打包进页面，离线也能看）。
  - ⇒ **视频不能内嵌 YouTube iframe**；改为"**YouTube 封面图卡片 + 点击跳转 YouTube 播放**"（成员均可访问，可接受）。
  - ⇒ 全部 CSS / JS 内联，无外部依赖。
- **响应式**：移动优先；页面主体**不得横向滚动**，宽内容（表格等）在自身容器内横向滚动。
- **主题**：浅色/深色自适应（`prefers-color-scheme` + 主题切换覆盖），两种主题都要清晰可读。

## 3. 内容模型

```
Attraction {
  id:        "A2"                    // 编号，投票用
  region:    "A"                     // 所属大区 A–E
  name_zh:   "乌布文化区"
  name_en:   "Ubud"
  tags:      ["文化","梯田","瀑布"]   // 标签 chips
  time:      "2–3 天"                // ⏱ 游玩时间
  price:     "门票各 ¥25–100；漂流约 ¥250–400；SPA ¥150+"   // 💰 人均
  highlight: true                    // 是否进"精华速览"
  subspots:  [ SubSpot, ... ]        // 招牌子景点（配图）
  video:     { yt_id, title }        // 1 个代表视频（封面卡）
  feature:   "…特点…"                // 可展开
  caution:   "…注意事项…"            // 可展开
  coupling:  "…耦合…"                // 可展开
}
SubSpot { name_zh, name_en, img /* data URI */, credit /* 图片来源/署名 */ }
```

## 4. 页面结构与 UX

1. **顶部 Header**：标题、行程窗口（10/2–10/11，约 9–10 天）、标签图例、A–E 快速跳转导航、一句"怎么用"。
2. **精华速览**：7 张大图卡（默认盘）。
3. **A–E 五大区**：每区一个 section header（区简介 + 建议天数 + 人均小计），下含若干**景点卡**：
   - 头部：编号+中英名、标签 chips、⏱时间、💰人均。
   - **媒体**：招牌子景点照片小图廊（点开 → 全屏 lightbox）+ 1 个视频封面卡（点开 → 跳 YouTube）。
   - **可展开详情**：特点 / 注意事项 / 耦合。
   - **投票控件**：`必去 / 可去 / 无所谓` 三态。
4. **组合建议 ①–④**：卡片式对比（移动端友好，宽表在容器内横向滚动）。
5. **参考价 & 通用注意事项**：折叠区。
6. **页脚**：图片来源/署名、数据免责声明、制作日期。
7. **悬浮"我的选择"入口**：显示已选数量，点开投票汇总面板。

**交互（纯原生 JS，无外部库）**：三态投票、localStorage 持久化、汇总生成与一键复制、图片 lightbox、平滑跳转导航、主题自适应。

## 5. "我的选择"投票功能（详细）

- **范围**：本地投票（每人在自己手机上，互不干扰）；**不做**云端 6 人票数汇总（Artifact 无后台）。
- **每张景点卡**：三态按钮 `必去 / 可去 / 无所谓`（默认无所谓/未选）。选择写入 `localStorage`。
- **顶部/面板可填**：
  - **名字**（文本框，存 localStorage）。
  - **主线偏好**：从 组①–④ 单选。
- **一键复制汇总**：点击生成如下文字并复制到剪贴板（`navigator.clipboard`，带 execCommand 兜底）：

  ```
  【我的印尼投票】@小李
  必去：A2 乌布、B1 佩尼达霸王龙、A1 乌鲁瓦图+火舞
  可去：A3 东巴厘火山、B2 蝠鲼浮潜
  主线偏好：组① 纯巴厘+佩尼达
  ```

  - 只列被标记为"必去""可去"的项；"无所谓/未选"不出现。
  - 名字为空时用 `@匿名`；主线未选时该行写"主线偏好：未选"。
  - 复制成功给出可见反馈（toast/按钮态）。
- **重置**：提供"清空我的选择"按钮。

## 6. 媒体来源、处理与授权

- **图片来源**：优先 **Wikimedia Commons（自由授权）** 等免费图源，直接取其指定宽度缩略图（约 1000px）。
  少数冷门点（如 Gamat Bay、Ceningan 蓝湖、Banyumala）若无好图源 → 用邻近代表图并注明，或略过该子景点。
- **视频来源**：为每个区/项搜到代表性 YouTube 视频，取 `img.youtube.com/vi/<id>/hqdefault.jpg` 官方封面做卡片，
  点击跳 `youtube.com/watch?v=<id>`；用 **oEmbed 校验**链接有效（已测通）。
- **准确性核对**：媒体清单单独成一份可核对数据（景点 → 图 URL / 视频 ID），**逐条核对"是不是这个地方"**，避免配错图误导投票。
- **内嵌与体积**：图片下载后转 data URI 内嵌；优先用图源自带的**定宽缩略图**控制尺寸（避免依赖本地压缩工具）。
  - **体积预算**：目标全页 **≤ ~6MB**；若超预算，按"减副图 → 降分辨率 → 减视频封面"顺序压缩。
  - 发布前**试发布一次**确认 Artifact 能正常加载、手机端流畅。
- **授权/署名**：页脚放"图片来源/署名"小节，列出 Commons 图片作者/许可；视频用官方封面+跳转（标准用法）。

## 7. 构建架构（数据驱动）

脚手架（不随成品交付，仅用于生成 `index.html`）：

1. **`data/attractions.json`** — 内容单一真源（第 3 节的结构 + 每个子景点的图源 URL、每项的 YouTube ID）。可复查。
2. **`scripts/fetch_media.py`** — 读 json 中的图源/视频 → 下载图片 + YouTube 封面 → 转 data URI → 产出 `media.json`（id → data URI）。
3. **`scripts/build.py`** — 用 `attractions.json` + `media.json` + 模板 → 渲染出**自包含** `index.html`。
4. **交付**：把 `index.html` 交给 Artifact 工具发布。

> 首个构建步骤先核查环境：Python 是否可用、是否需要 `pip install` 压缩库（尽量靠定宽缩略图避免）。
> 实现 UI 前须加载 `artifact-design` 与 `frontend-design` skill 以定调视觉。

## 8. 范围与非目标

**做**：
- 15 个景点项 + 精华速览 + 组合 ①–④ + 参考价 + 通用注意事项。
- 每项真实照片 + 视频封面卡。
- 本地"我的选择"投票（含名字 + 主线①–④）+ 一键复制。
- 移动优先、深浅色自适应、自包含单页。

**非目标（YAGNI）**：
- ❌ 真正的 6 人云端票数自动汇总（无后台；用户未选此项）。
- ❌ "本次不纳入"的远程项目（日惹、布罗莫、四王群岛、苏拉威西等）——**整节删除，不在页面出现**。
- ❌ 实时价格/航班/开放时间抓取（价格沿用源文档的粗略估算并注明）。

## 9. 风险与缓解

| 风险 | 缓解 |
|---|---|
| 媒体逐条核对最耗时、易配错图 | 图源/视频集中在可复查的 `attractions.json`；逐条核对地点；oEmbed 校验视频 |
| data URI 内嵌导致页面偏大 | 用定宽缩略图；精选 45–50 图；体积预算 ≤6MB；试发布验证 |
| 少数冷门点无好免费图源 | 用邻近代表图并注明，或略过该子景点（不影响主景点） |
| 图片版权/署名 | 优先 Commons 自由授权；页脚署名；视频用官方封面+跳转 |
| Artifact 尺寸/加载上限不确定 | 先小规模试发布探边界，再决定最终图量/分辨率 |
