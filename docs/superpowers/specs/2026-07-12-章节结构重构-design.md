# 网页章节结构重构 · 设计说明

> **目标**：把当前「精选（主题）＋ Region A–F（地理）」混编的章节，改成 **第一章＝最热门景点，其后按地区依次成章** 的清爽结构；顺带消除重复面、空壳章与被掏空的区域。
>
> 日期：2026-07-12 · 状态：待实施

---

## 1. 背景 · 现状问题

页面滚动顺序由 `webpage/build.py` 第 253 行 `BODY` 组装。当前存在四个问题：

1. **命名混搭** — 章节是「精选（主题）＋ Region A–F（地理）」混编，没有清爽的「第一章/第二章」序号；A–F 地理编号又暗示"严格按地区"，与实际内容（热门景点已前置）不符。**这是本次重构的直接动因。**
2. **两个重复面** — 顶部「精华速览」横滑卡与「精选景点」章展示同一批 5 个景点，且紧挨着。
3. **空壳章** — 科莫多、布罗莫两整章没有景点卡（各自只有一个景点，且都被当作热门抽到前面），只留一句"已在上方精选展示"。
4. **区域被掏空** — 佩尼达一章的两张招牌被抽走，只剩蓝梦岛。

---

## 2. 关键决策

1. **结构模型：抽出 · 不设空章。** 热门景点只在第一章出现；整区只有一个热门景点的科莫多、布罗莫**不再单列区域章**（已完整收录在第一章）。→ 无空壳章、无重复投票卡。
2. **删除顶部「精华速览」横滑 shelf。** 它与第一章内容重复；第一章本身就在最顶、标题即「最热门景点」，已承担"先看这一屏"；跳转导航交给 hero 顶部那条。
3. **第一章成员＝4 个**：乌布(A1)、佩尼达悬崖(B1)、科莫多船宿(C1)、布罗莫日出(D1)。**佩尼达浮潜(B2) 移出第一章**，回到第三章「佩尼达三岛」。★"精华"星标此后精确表示"第一章成员"。

## 4. 详细改动

### 4.1 `webpage/content.py`（小改）
- `HIGHLIGHTS` 去掉 B2 条目。删除 shelf 后，`HIGHLIGHTS` 的 `img`/`title`/`blurb` 字段不再被任何函数使用；将其简化为有序 id 列表（如 `HIGHLIGHTS = ["A1", "B1", "C1", "D1"]`），`build.py` 里 `HL_IDS` 相应改为直接引用。
- B2（`id:"B2"`）去掉 `"highlight": True`，使 ★ 星标与"第一章成员"一致。
- 其余不动：REGIONS 仍标 A–F，ITEMS 顺序不变，图片 key 不变。

### 4.2 `webpage/build.py`（主要改动）
- **删除 `highlights()`**（横滑 shelf 函数）及其在 `BODY` 中的调用。
- **`region()`**：当区域过滤后无景点时**返回空串**（整章跳过），删除"该区域景点已在上方…"空壳分支及 `chap-empty` 相关输出。
- **章节序号机制**：按实际渲染顺序生成"第 N 章"。景点章序列＝`[第一章 最热门]` ＋ `[REGIONS 中非空的区域，按原顺序]`。
- **章节标题与副标题（eyebrow）**：每个景点章的 `.chap-region` eyebrow 由原来的 "Region A" / "Highlights" 统一改为 **"第 N 章"**（这是本次要的「第一章/第二章」框架落点）；`.chap-meta h2` 标题保持各章名称——第一章为「最热门景点」，其余章沿用 `REGIONS` 中的名称（本就含 "(Bali)" 等英文，无需另加）。
- **`hero()` 导航条**：从上面这份"实际渲染章节列表"生成（第一…第五章＋锚点），不再硬编码 A–F；锚点沿用 `#hl-main`（第一章）与 `#region-{id}`（区域章），保证与正文一一对应，且日后调整热门集合时导航自动跟随。
- **`highlights_main()`**：标题（h2）改为「最热门景点」、eyebrow 为「第一章」，仍从 `HL_IDS` 取项渲染完整投票卡。
- 组合建议 / 价格 / 注意事项 / 页脚：保持**不编号**的收尾区，逻辑与样式不变。

### 4.3 `webpage/style.css`（可选小改）
- 若「第 N 章」eyebrow 需与原 "Region A" 视觉保持一致，做小幅样式调整；无结构性改动。
- `.chap-empty`（空壳章提示）样式随分支删除而弃用，可一并清理。

## 5. 一个小取舍

科莫多、布罗莫的**区域级概览行**（建议天数 / 人均预算 / 区域描述，来自 `REGIONS` 的 `days`/`budget`/`desc`）随区域章取消而不再单独显示；两者各自的景点卡内仍保留「时间 / 人均」。

- **默认**：接受此取舍（信息主要在景点卡内，损失有限）。
- **可选补救**（实施时若你要求）：把这两句概览并入第一章对应卡片的说明，做到信息零丢失。

## 6. 明确不做（YAGNI）

- 不改投票逻辑（`app.js`）、云同步（`cloud` 模块）、汇总算分（`tally.js`）。
- 不改图片、不重抓媒体、不新增任何依赖（仍只用 Python 标准库）。
- 不改组合线路 / 价格 / 注意事项的**内容**（仅它们在页面中保持不编号）。

## 7. 验收标准

1. `cd webpage && python3 build.py` 成功、无报错，正常写出 `index.html` 与 `images/`。
2. `node --test webpage/test/tally.test.js` 通过（本改动不触及汇总，应不受影响）。
3. 本地打开 `webpage/index.html` 逐项核对：
   - 章节顺序＝第一章…第五章；
   - 第一章 4 张卡（乌布 / 佩尼达悬崖 / 科莫多 / 布罗莫）；
   - 第三章「佩尼达三岛」含 蝠鲼&海龟浮潜 ＋ 蓝梦岛；
   - **无**科莫多 / 布罗莫空壳章；
   - **无**顶部「精华速览」横滑卡；
   - hero 顶部跳转条＝第一章…第五章，点击可正确跳转到各章。
4. 通过后按 `CLAUDE.md` 部署 `index.html`（图片无变化，无需部署 `images`）。
