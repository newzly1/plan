## 一、NoSQL 文档数据库（已开通，可直接用）

你的环境已经有一个运行中的文档数据库实例，无需额外开通即可使用。这是 CloudBase 最核心的后端能力之一。

### 能力概述

文档数据库是基于 MongoDB 协议的 NoSQL 数据库，以集合（Collection）和文档（Document）的形式组织数据。每个文档是一个 JSON 对象，支持灵活的 schema。核心能力包括：CRUD 操作、复杂条件查询（where、逻辑运算、正则匹配）、聚合管道（aggregation）、地理位置查询（near、within）、排序分页等。

### 实时数据同步

文档数据库支持 watch 机制，客户端可以订阅某个集合或查询的数据变化。当数据库中的数据发生增删改时，所有订阅了该数据的客户端会实时收到推送。体验版支持 10 个并发实时推送连接。这个能力非常适合做协作工具——比如多人同时编辑同一个旅行清单时，每个人的修改都能实时同步给其他人。

### 权限控制

通过安全规则系统实现细粒度权限控制。每条数据默认携带 `_openid` 字段（创建者的身份标识），可以配置安全规则让用户只能读写自己的数据，或实现更复杂的权限逻辑（如管理员可读写全部、普通用户只能读公开数据）。

### 对你的旅行项目的意义

当前你的投票网页是纯前端 localStorage 方案，数据只存在各自的浏览器里。如果接入数据库，可以实现：多人投票结果实时汇总（所有人看到同一份投票统计）、行程清单云端持久化（换设备也能看到自己的选择）、管理员后台管理景点数据（不用改 content.py 重新构建）。

## 二、MySQL 关系型数据库（可开通）

CloudBase 同时提供 MySQL 关系型数据库，适合需要复杂关联查询、事务支持的场景。体验版支持关系型数据库功能。与 NoSQL 的区别在于：MySQL 适合结构化数据、多表关联、事务一致性要求高的业务（如订单系统、财务系统）；NoSQL 适合灵活 schema、快速迭代的业务（如内容管理、用户配置）。两者可以同时使用，按需选择。

开通方式是在 CloudBase 控制台的数据库管理页面点击启用 MySQL，之后可以通过 Web SDK（`app.rdb()`）、云函数中的 Node SDK、或 HTTP API 进行操作。支持标准的 SQL 语法，每张表需包含 `_openid` 字段用于行级权限控制。

## 三、云函数（命名空间已就绪，可直接部署）

你的环境已有云函数命名空间，目前未部署任何函数。云函数是 Serverless 计算服务，无需管理服务器，上传代码即可运行，按调用次数和执行时间计费。

### 支持的运行时

Node.js（主力支持，版本 18/20）、Python（3.9/3.12）、PHP、Java、Go。其中 Node.js 生态最完整，支持最多内置模块和 SDK 集成。

### 触发方式

HTTP 触发：云函数可以暴露为一个 HTTP 接口，前端通过 fetch/axios 直接调用，适合构建 API 服务。

定时触发器：按 cron 表达式定时执行，适合定时任务（如每天凌晨清理过期数据、定时抓取汇率更新旅行费用估算）。

数据库触发：当数据库发生增删改时自动触发，适合数据联动（如用户投票后自动更新统计缓存）。

云函数间调用：一个函数可以调用另一个函数，实现函数编排。

### 关键限制

体验版调用次数配额为每月 20 万次，单次执行超时最大 900 秒，内存可自定义配置。不支持连接腾讯云 VPC 和固定出口 IP（这些是标准版及以上才支持的功能）。

### 对你的旅行项目的意义

可以用云函数实现：投票统计 API（接收前端投票、写入数据库、返回实时统计）、汇率转换服务（定时抓取印尼盾汇率供费用估算调用）、AI 行程规划接口（在云函数中调用 CloudBase AI 模型，根据用户选择的景点生成行程建议）。

## 四、云托管 CloudRun（可开通）

云托管是比云函数更重量级的计算服务，支持运行完整的容器化应用或常驻服务。你的环境目前没有云托管服务，但可以随时创建。

### 两种模式

容器型（推荐）：支持任意语言和框架（Java/Go/Python/PHP/.NET/Node.js 等），通过 Dockerfile 构建镜像部署，适合完整的后端服务、长连接应用、AI Agent 服务。支持自动扩缩容、灰度发布、最小实例数配置（减少冷启动）。

函数型：仅支持 Node.js，有特殊限制，适合简单的 API 服务。本质上是以云托管的方式运行函数。

### 与云函数的区别

云函数是事件驱动的、短暂的、无状态的计算单元，适合单次请求处理；云托管是常驻的、有状态的服务，适合需要长连接（WebSocket/SSE）、需要保持内存状态（如 AI 模型加载）、需要完整框架（如 Express/Koa/NestJS）的场景。云托管可以设置最小实例数为 1 来避免冷启动，云函数则不可避免。

### 对你的旅行项目的意义

如果你要做一个 AI 旅行规划助手（流式对话、需要保持上下文），云托管比云函数更合适，因为流式响应需要长连接。如果你要部署一个完整的管理后台（带认证、数据库 CRUD、文件上传），云托管可以跑一个完整的 Express/NestJS 应用。

## 五、云存储（已开通，可直接用）

你的环境已有一个云存储桶（CDN 域名 `706c-plan-d0gstt7r6507aa319-1451599494.tcb.qcloud.la`），目前未使用。云存储用于存放和管理任意文件，与静态托管的区别在于：静态托管专为部署前端网站设计（放 HTML/CSS/JS），云存储是通用文件存储（放用户上传的图片、文档、视频等）。

### 核心能力

### 对你的旅行项目的意义

## 六、身份认证（可启用）

CloudBase 内置完整的用户身份认证系统，支持多种登录方式，无需自己实现注册登录逻辑。

### 支持的登录方式

Web 端：邮箱密码、手机号验证码、微信扫码登录（需公众号）、GitHub/Google/Apple 等 OAuth 登录、匿名登录。小程序端：微信登录（免鉴权，自动获取 openid）。每种登录方式都需在控制台手动开启并配置对应参数。

### 用户管理

认证系统提供完整的用户管理能力：用户注册/登录/登出、用户信息查询和更新、密码重置、账号绑定/解绑、Token 管理和刷新。每个用户有唯一的 uid，数据库中的数据通过 `_openid` 字段与用户关联，实现行级数据隔离。

### 权限联动

认证系统与数据库、云存储的安全规则深度联动。可以配置规则如"用户只能读写自己创建的数据""管理员可读写全部数据"等，权限在数据层面自动生效，无需在业务代码中手动判断。

### 对你的旅行项目的意义

当前投票网页是匿名的，6 个人各自的投票结果存在各自的浏览器里。接入认证后可以实现：每个投票者有明确身份（不用再问"这是谁投的"）、投票数据与用户绑定存储到云端、管理员可以看到每个人的投票明细、防止重复投票。

## 七、AI 大模型调用（可启用，重点能力）

CloudBase 内置了统一的大模型接入能力，这是体验版最具价值的高级功能之一。你可以在网页或云函数中直接调用 DeepSeek、混元等大模型，无需自己去各模型平台申请 API Key。

### 内置模型（无需额外配置）

混元（hunyuan-exp）：腾讯自研大模型，使用 CloudBase 共享 Token 配额，无需单独申请。小程序成长计划可免费领取 1 亿 Token。模型 ID 包括 `hunyuan-2.0-instruct-20251111`、`hunyuan-turbos-latest` 等。

DeepSeek：使用 CloudBase 共享 Token 配额。模型 ID 包括 `deepseek-v4-flash`、`deepseek-v3.2` 等。

### 自定义模型（需配置 API Key）

支持接入兼容 OpenAI 协议的第三方平台：DeepSeek 开放平台、智谱 AI（GLM）、硅基流动（可间接接入 Kimi）、火山方舟、阿里云百炼、百度云千帆、讯飞星辰等。模型 ID 包括 `glm-5`、`kimi-k2.6` 等。

### 核心能力

文本生成（generateText）：一次性返回完整结果，适合短文本生成。

流式文本生成（streamText）：逐字返回，适合对话场景和长文本生成，用户体验更好。

图片生成（generateImage）：文生图，仅 Node SDK 支持（需在云函数或云托管中调用），模型为 `hunyuan-image`。

图片理解：图生文，分析图片内容。

工具调用（Function Calling）：让 AI 调用外部工具，适合构建 AI Agent。

### 调用方式

Web 端使用 `@cloudbase/js-sdk`，通过 `app.ai().createModel().streamText()` 直接在浏览器中调用。Node.js 端使用 `@cloudbase/node-sdk`，在云函数或云托管中调用。小程序端使用 `wx.cloud.extend.AI`。也支持 HTTP API 和 OpenAI SDK 兼容调用。

### 计费

内置模型使用共享 Token 配额包。各付费套餐均含 10 万免费 Token。参与 AI 小程序成长计划可免费获得 1 亿混元 Token 和 1 万张混元生图。体验版的 Token 额度需确认控制台具体配置。

### 对你的旅行项目的意义

这是最令人兴奋的能力。你可以在旅行网页中直接集成 AI 旅行助手：根据用户选中的景点，AI 自动生成行程规划建议（"我选了布罗莫火山和伊真火山，帮我安排 3 天行程"）；AI 智能问答（"巴厘岛 7 月天气怎么样？需要带什么？"）；根据预算和天数自动优化行程；甚至用图片生成能力为行程生成配图。

## 八、微信生态集成

CloudBase 与微信生态深度集成，这是它相比其他云平台最大的差异化优势。

### 微信小程序开发

CloudBase 是微信小程序云开发的底层基础设施。你可以在小程序中直接使用 `wx.cloud` API 访问数据库、云函数、云存储，无需额外初始化。小程序登录是免鉴权的，自动获取用户 openid。

### 公众号集成

支持微信公众号接入，包括 OAuth 授权获取用户信息、JSAPI 支付、公众号消息推送、公众号客服消息。可以通过云函数处理公众号事件。

### 微信支付

CloudBase 集成中心提供微信支付相关云函数模板，支持小程序支付、公众号 JSAPI 支付、Native 扫码支付。支付回调可通过云函数处理。

### 对你的旅行项目的意义

如果你想把旅行投票工具升级为微信小程序，CloudBase 是最省事的选择——前后端一体化，登录免鉴权，数据库和存储直接用。6 个人在微信里投票比网页更方便。

## 九、其他能力

### 微搭低代码平台

CloudBase 内置微搭低代码开发平台，支持可视化拖拽搭建应用，适合不写代码的业务人员快速搭建管理后台、表单应用、数据看板。体验版可发布 3 个应用。

### 实时通信

通过数据库 watch 机制、云托管 WebSocket、SSE 流式响应等方式实现实时通信。适合聊天、协作编辑、实时通知等场景。

### 数据模型管理

控制台提供可视化的数据模型管理，可以图形化地设计数据库结构、定义字段类型、设置关联关系，自动生成数据库集合和 CRUD 接口。

### 日志与监控

所有云函数调用、数据库操作、存储访问都有日志记录，控制台提供调用次数、错误率、响应时间等监控指标。

### 集成中心

提供常用第三方服务的集成模板（微信支付、企业微信、短信通知、邮件发送等），一键创建对应的云函数。

## 综合分析与建议

你的 CloudBase 体验版实际上是一个完整的后端一体化平台，目前只用了不到十分之一的能力（仅静态托管）。以下是按实用程度排序的建议：

### 立即可做（零额外成本）

AI 旅行助手集成：在你的网页中引入 `@cloudbase/js-sdk`，调用内置的混元或 DeepSeek 模型，让用户可以与 AI 对话规划行程。这是最惊艳的升级，且内置模型使用共享 Token 配额。

数据库投票汇总：创建一个 `votes` 集合，用户投票时写入数据库，所有人看到实时汇总结果。用 watch 机制实现投票数实时更新。

### 进阶可做（需少量开发）

身份认证 + 云端清单：开启邮箱或匿名登录，用户的"我的清单"存到数据库而非 localStorage，换设备也能查看。

云函数 API：部署一个云函数处理投票逻辑（防重复投票、统计汇总、数据校验），前端通过 HTTP 调用。

定时汇率更新：用定时触发器云函数每天抓取印尼盾汇率，供网页费用估算使用。

### 需要升级才能做的

云托管 AI Agent：如果要做复杂的流式 AI 对话（需要长连接、保持上下文），建议升级到标准版后用云托管部署。

微信小程序版：体验版可以开发小程序，但如果发布上线需要转为付费环境。

高并发场景：体验版 3000 资源点/月和 500 QPS 限制，适合开发测试和小规模使用（6 人投票完全够用），但不适合面向公众的高并发应用。

### 体验版限制提醒

体验版每月 3000 资源点（涵盖数据库读写、云函数调用、存储、CDN 等所有资源消耗），不支持加购资源包和按量付费，2027-01-08 到期需手动续期。对于你的 6 人旅行投票项目来说，这些配额绰绰有余。但如果要长期面向更多用户运营，建议升级到个人版（19.9 元/月）获得 3GB 容量和 20 万次调用。
